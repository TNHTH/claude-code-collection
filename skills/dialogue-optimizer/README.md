# Dialogue Optimizer - 快速指南

> **自动评估和优化每次对话的效率**
> **无需用户提醒，自动执行**

---

## 🎯 这个Skill做什么？

```
每次对话结束时自动运行，分析：
✓ Token使用效率
✓ 输出质量评估
✓ 工具调用优化
✓ 生成改进建议

目标：用最少的token，做到最好的效果
```

---

## 🚀 自动执行流程

```
对话结束
   ↓
自动触发评估
   ↓
收集数据（token使用、工具调用等）
   ↓
计算效率指标
   ↓
识别问题点
   ↓
生成改进建议
   ↓
应用到下次对话
   ↓
持续优化
```

---

## 📊 评估维度

### 1. Token效率
```
输出/输入比 > 3:1 ✅
单次输出 < 2000 tokens ✅
重复内容 < 10% ✅
```

### 2. 信息密度
```
实质性内容 > 70% ✅
完整性 > 90% ✅
准确性 > 95% ✅
```

### 3. 工具效率
```
成功率 > 95% ✅
并行率 > 50% ✅
无冗余调用 ✅
```

### 4. 结构化输出
```
优先使用表格 ✅
使用列表 ✅
提供引用 ✅
```

---

## 📝 输出内容

每次对话后生成：

1. **SESSION_YYYY-MM-DD.md**
   - 本次对话的详细评估
   - 问题识别
   - 改进措施
   - 下次优化目标

2. **OPTIMIZATION_LOG.md**
   - 长期优化记录
   - 效果跟踪
   - 最佳实践库

3. **自动应用改进**
   - 下次对话自动应用
   - 无需用户提醒

## 🎯 核心优化策略

### 策略1：结构化优先

```markdown
❌ 不好：用3段文字描述5个选项

✅ 好：用1个表格展示5个选项

效果：token减少60%，可读性提升
```

### 策略2：并行调用

```markdown
❌ 不好：
  Read file1 → Read file2 → Read file3

✅ 好：
  并行调用 Read file1, file2, file3

效果：时间减少70%
```

### 策略3：引用链接

```markdown
❌ 不好：详细解释MCP协议（500字）

✅ 好：参考[MCP协议文档](link)（10字）

效果：token减少80%
```
